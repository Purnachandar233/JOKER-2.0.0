JOKER Music Bot Privacy Policy

How we collect, use, share, and delete data.

Last Updated: March 23, 2026
Policy Version: 2.0

1) Overview

This policy applies to JOKER Music Bot ("JOKER", "bot", "we", "our"). It explains what data we process when you use bot commands in Discord and what controls you have.

2) Data We Process

Data Type Why It Is Used Stored In
Discord User ID Account-scoped features such as profile stats, premium windows, and moderation flags. MongoDB
Guild/Server IDs and channel/role IDs Server settings such as welcome configuration, requester mode, DJ role, and prefix settings. MongoDB
Command usage stats Track feature usage and profile counters (for example: commands used, songs listened, listen time). MongoDB
Music search queries and track metadata Find and play requested songs via audio/search providers. Lavalink + temporary runtime memory
Vote and premium state Grant and validate vote/premium gated features. MongoDB + Top.gg API checks
Error and diagnostic logs Operational monitoring, debugging, and abuse handling. Console + in-memory buffers

3) Message Content and Privileged Intents

JOKER uses Discord privileged intents, including Message Content and Guild Members. Message content is processed to run prefix commands and mention-prefix command parsing. Slash command usage does not require message text parsing.

4) What We Do Not Collect

  • Discord passwords or authentication credentials.
  • Payment card information.
  • Private DMs unrelated to bot interactions.
  • Your IP address directly from Discord clients.

5) Data Sharing and Third Parties

We only share data when needed for bot operation:

  • Discord API: receives command interactions and message events required for bot functionality.
  • MongoDB: stores persistent bot settings and user records.
  • Lavalink / audio sources: receives search terms and track metadata to resolve music playback.
  • Top.gg: used to validate voting status and vote-based premium windows.

We do not sell personal data.

6) Retention

Category Retention
User profile stats Kept until user deletion request or internal cleanup decisions.
Vote-based premium window 12 hours from vote grant unless replaced by permanent premium.
Guild settings Kept while bot is configured in that guild, or until removed/reset.
In-memory recent logs Runtime-only; cleared on process restart (default buffer: 500 entries).

7) Your Rights and Controls

You can request data access, correction, or deletion through our support channels:

  • Support server: discord.gg/JQzBqgmwFm
  • Privacy email: privacy@jokerbot.com
  • /privacy and /terms are always available for legal links.
We process verified privacy requests as quickly as possible. Third-party platform logs are managed by their respective providers.

8) Security Practices

  • Secrets should be provided through environment variables.
  • Error sanitization is applied to reduce accidental credential leakage in logs.
  • Access controls are enforced for owner-only and permission-gated commands.
